Editing Lifecycle Methods

textFieldShouldBeginEditing(_:)

Before becoming the first responder, the text field calls its delegate's textFieldShouldBeginEditing(_:) method. Use it to allow or prevent the editing of the text field's contents.

textFieldDidBeginEditing(_:)

Called as soon as the user taps inside the text field to edit.

textFieldShouldEndEditing(_:)

Before resigning as first responder, this method is called. Use this method to validate the current input.

textFieldDidEndEditing(_:)

This method is called after the text field resigns as first responder.

The Primary Method for Controlling Text

textField(_:shouldChangeCharactersIn:replacementString:)

This method is called whenever the current text changes. You can do what you need to, then return true if you want to allow the text field to change its text.

The Clear Button (The x on the right side of some text fields

textFieldShouldClear(_:)

This method is called when the user taps the built-in button to clear the text.

The Return Key in the Keyboard

textFieldShouldReturn(_:)

This method is called then the user taps the keyboard's return button. You can implement this to dismiss the keyboard on pressing return. This can be done with textField.resignFirstResponder(), and then returning true.
